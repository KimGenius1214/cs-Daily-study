📊 통계·지표 카드 (Stat / KPI Card) 데이터·테이블·차트

대시보드 맨 위에 놓이는 "숫자 한 개 + 증감 + 미니 추세" 카드. 결제 대시보드의 오늘 거래액·승인율·환불 같은 핵심 지표를 한눈에 보여준다. 값만 크게 쓰고, 전 기간 대비 변화(델타)를 색과 화살표로, 흐름을 스파크라인(축·라벨 없는 미니 라인차트)으로 압축한다.

데이터 도착 전에는 스켈레톤으로 레이아웃 점프(CLS)를 막는다.
오늘 거래액
₩ 48,210,000
12.4%어제 대비
결제 승인율
97.3%
0.6%p지난주 대비
환불 건수
37건
8.1%어제 대비

▲/▼ + 색으로 "좋음/나쁨"을 판단하되, 환불 건수처럼 감소가 좋은 지표는 화살표 방향(감소=아래)과 의미(초록=좋음)를 분리해서 설계해야 오해가 없다. 여기선 "환불 감소"라 아래 화살표+빨강 대신 맥락상 검토가 필요함을 보여주려 빨강으로 뒀다 — 실제로는 지표별로 색 규칙을 정의하는 게 안전하다.

<article class="stat pos">
  <div class="label"><span class="dot"></span>오늘 거래액</div>
  <div class="value">₩ 48,210,000</div>
  <span class="delta up">▲ 12.4%</span> <span class="vs">어제 대비</span>
  <svg class="spark" viewBox="0 0 66 26"><path d="M1 20 L31 12 L65 3"/></svg>
</article>

/* 값은 tabular-nums 로 자릿수 흔들림 방지 */
.stat .value{ font-variant-numeric: tabular-nums; }
/* 스켈레톤: 데이터 오기 전 자리를 미리 잡아 레이아웃 점프 차단 */
.stat.loading .value{ /* 배경 shimmer */ }

🔔 인라인 배너·알럿 (Inline Alert) 피드백·상태

토스트(잠깐 떴다 사라지는 알림)와 달리 인라인 배너는 화면 흐름 안에 계속 머무는 상태 메시지다. "카드 만료 임박", "정산 실패 — 재시도" 같은 지속적·조치가 필요한 정보에 쓴다. 4가지 톤(정보·성공·경고·위험)을 색·아이콘·역할(role)로 구분하고, 필요하면 액션 버튼과 닫기를 붙인다.

새 정산 리포트가 준비됐어요

6월 정산 내역을 확인할 수 있습니다.

결제 수단이 등록됐습니다

기본 카드로 설정되었어요.

카드 만료 임박

등록된 카드가 이번 달 만료됩니다. 미리 갱신하세요.

정산 이체에 실패했어요

일시적 오류입니다. 잠시 후 다시 시도해 주세요.

role 구분이 핵심: 정보·성공·경고는 role="status"(=aria-live="polite", 하던 일 끝나면 읽어줌), 즉시 주의가 필요한 위험은 role="alert"(=aria-live="assertive", 바로 끊고 읽어줌). 색만으로 구분하지 말고 아이콘·제목 텍스트로도 의미를 이중 전달해야 색맹 사용자도 안다.

<!-- 위험: 즉시 읽어줌 -->
<div class="alert danger" role="alert">
  <svg class="ic" ...></svg>
  <div class="txt">
    <p class="ttl">정산 이체에 실패했어요</p>
    <p class="msg">잠시 후 다시 시도해 주세요.</p>
    <button class="btn brand">다시 시도</button>
  </div>
  <button class="x" aria-label="닫기">×</button>
</div>

/* 정보/성공/경고 → role="status" (polite),  위험 → role="alert" (assertive) */

🌀 @starting-style — JS 없이 "나타나는" 애니메이션 트렌드·개념

토스트·모달·배너를 부드럽게 등장/퇴장시키려면, 예전엔 display:none → block 사이에 setTimeout으로 클래스를 토글하는 해킹이 필요했다. 2024년 Baseline(주요 브라우저 공통 지원)에 오른 @starting-style + transition-behavior: allow-discrete 덕에, 이제 CSS만으로 display 전환까지 트랜지션을 걸 수 있다. (Baseline = 크롬·사파리·파이어폭스에서 공통으로 쓸 수 있게 된 기능이란 뜻.)

저장되었습니다

display:none → flex 전환까지 CSS만으로 애니메이션

등장·퇴장 모두 CSS 트랜지션. JS는 data-shown 속성만 바꾼다 — 타이머로 클래스를 조율하던 코드가 사라진다.

브라우저 지원(2024-08 Baseline Newly available): Chrome 117+ 지원 · Safari 17.5+ 지원 · Firefox 129+ 지원 — 미지원 브라우저에선 애니메이션만 생략되고 표시/숨김은 정상 동작(점진적 향상).
/* 1) 평소 상태(보일 때) */
.toast{
  opacity:1; transform:translateY(0);
  transition: opacity .3s, transform .3s,
              display .3s allow-discrete;   /* display 도 트랜지션 대상에 */
}
/* 2) 숨김 상태 */
.toast[data-shown="false"]{
  opacity:0; transform:translateY(12px);
  display:none;                              /* discrete 라 보통은 즉시 사라짐 */
}
/* 3) 등장 직전의 시작값 — 이게 핵심 */
@starting-style{
  .toast[data-shown="true"]{ opacity:0; transform:translateY(12px); }
}

// JS 는 딱 이것뿐
btn.onclick = () => toast.dataset.shown = "true";
  • allow-discrete: display처럼 중간값이 없는(끊기는) 속성도 트랜지션 타이밍에 맞춰 바뀌게 해주는 키워드.
  • @starting-style: 요소가 처음 나타날 때의 "출발 지점" 스타일. 이게 없으면 진입 애니메이션이 안 걸린다.
  • 모달·팝오버(top layer)에도 동일하게 적용 가능. prefers-reduced-motion 존중은 별도로 챙길 것.
