🪟 모달 다이얼로그 — native <dialog> · 포커스 트랩 · 스크롤 잠금 오버레이·모달·토스트

결제 확인처럼 "꼭 답해야 넘어가는" 흐름은 모달 다이얼로그로 막는다. 2026년엔 직접 div로 만들지 않고 브라우저 기본 <dialog> 요소를 쓴다. showModal() 한 줄이면 포커스 트랩(탭이 모달 밖으로 안 나감), ::backdrop(뒤 어둡게), ESC 닫기, 배경 inert 처리를 브라우저가 알아서 해준다.

버튼을 누르면 모달이 부드럽게 열립니다. Tab으로 돌려보면 포커스가 모달 안에만 머뭅니다. ESC 또는 배경 클릭으로 닫힙니다(스크롤도 잠깁니다).

  • 포커스 트랩 = 모달이 열린 동안 Tab 키가 모달 안 요소만 순환하는 것. 키보드 사용자가 보이지 않는 배경으로 빠지지 않게 한다.
  • inert = 해당 영역을 클릭·포커스·스크린리더 대상에서 통째로 빼는 속성. showModal()이 배경에 자동 적용한다.
  • paysable 적용: 결제 확인·3DS 인증·세션 만료 안내처럼 차단형(blocking) 흐름에. 토스트(비차단)와 역할이 다르다.

🔔 진행 표시 — 멀티스텝 스텝퍼 · 결정형/비결정형 프로그레스 피드백·상태(로딩·빈상태·에러)

"지금 어디쯤이고, 얼마나 남았나"를 보여주는 게 체크아웃 이탈을 줄인다. 단계가 정해진 흐름은 스텝퍼, 끝나는 시점을 아는 작업은 결정형(determinate) 막대, 모르는 작업은 비결정형(indeterminate) 막대로 구분한다.

완료된 단계는 ✓로 채워지고 현재 단계는 링으로 강조됩니다. 숫자가 아니라 상태를 보여주는 게 핵심.

결정형 막대 — 진행률을 알 때
0%
비결정형 막대 — 끝을 모를 때

끝나는 시점을 모르면 가짜 %를 보여주지 말 것. 움직이는 막대로 "살아있다"만 전한다.

  • 결정형 vs 비결정형: 남은 시간을 계산할 수 있으면 결정형(%), 없으면 비결정형(무한 애니메이션).
  • 접근성: 진행 막대엔 role="progressbar" + aria-valuenow로 실제 값을 노출한다.
  • paysable 적용: 체크아웃 상단 스텝퍼 + 결제 승인 대기(PG 응답)엔 비결정형. prefers-reduced-motion이면 애니메이션을 끈다.

🌀 @starting-style — display:none에서도 부드럽게 열고 닫기 트렌드·개념

오랫동안 display:none이나 <dialog>·팝오버의 등장/퇴장 애니메이션은 JS 타이머로 클래스를 붙였다 떼며 흉내 냈다. 2026년엔 CSS만으로 된다. @starting-style(나타나기 직전의 "시작 상태"를 정의)과 transition-behavior: allow-discrete(display 같은 불연속 속성도 전환에 참여)를 함께 쓰면 된다. 위 모달·아래 토스트가 모두 이 방식으로 움직입니다.

JS는 data-show 속성만 토글합니다. 나타나고 사라지는 모션은 전부 CSS가 담당합니다.

/* 1) 평소(닫힘) → 보일 때의 목표 상태 */
.toast { opacity:0; transform:translateY(12px);
  transition: opacity .25s, transform .25s,
              overlay .25s allow-discrete,
              display .25s allow-discrete; }
.toast[data-show="true"] { opacity:1; transform:none; }

/* 2) "나타나기 직전" 시작 상태 — 이게 없으면 등장 모션이 안 돈다 */
@starting-style {
  .toast[data-show="true"] { opacity:0; transform:translateY(12px); }
}
  • 왜 지금: Firefox 129로 @starting-style·allow-discrete가 Baseline(주요 브라우저 모두 지원, 2026 기준)에 들어왔다. Chrome 117+/Safari 17.5+/Firefox 129+.
  • graceful degradation = 구형 브라우저에선 모션만 생략되고 요소는 정상적으로 보였다 사라진다(기능은 안 깨짐).
  • paysable 적용: 토스트·모달·드롭다운의 등장/퇴장에서 Framer Motion 같은 JS 라이브러리 의존을 줄일 수 있다(번들↓).
