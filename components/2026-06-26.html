📊 데이터 테이블 — 정렬·고정 헤더·행 선택 데이터·테이블·차트

결제 백오피스의 거래 테이블. 헤더 클릭으로 정렬하고(키보드 Enter/Space도 동작), 스크롤해도 헤더가 고정되며, 체크박스로 행을 선택합니다. 금액은 우측 정렬 + tabular-nums(숫자 폭 고정)로 자릿수가 어긋나지 않게 했습니다.

거래 내역. 열 제목 버튼을 누르면 정렬됩니다.
행을 선택하면 여기에 개수가 표시됩니다.

왜 이렇게? 헤더는 <th> + <button>이고 정렬 상태를 aria-sort(none/ascending/descending)로 알려 스크린리더가 "오름차순 정렬됨"을 읽습니다. 화살표는 색뿐 아니라 모양(▲▼)으로도 방향을 구분합니다.

🔔 토스트 알림 — 스택·자동 닫힘·액션·접근성 피드백·상태

작업 결과를 화면 모서리에 잠깐 띄우는 비모달(non-modal: 화면을 막지 않는) 알림입니다. 여러 개가 쌓이고, 일정 시간 뒤 자동으로 닫히되 마우스를 올리면 타이머가 멈춥니다(읽을 시간 확보). 스크린리더에는 aria-live 영역으로 자동 전달됩니다.

왜 이렇게? 일반 알림은 role="status"(= aria-live="polite", 사용자가 하던 걸 방해하지 않고 알림)로, 에러는 즉시 알려야 하므로 role="alert"(assertive)로 띄웁니다. 하단 진행바가 남은 시간을 보여주고, hover 시 멈춥니다.

🌀 Optimistic UI — 응답을 기다리지 않고 먼저 반영하기 트렌드·개념

Optimistic UI(낙관적 업데이트): 서버 응답을 기다리지 않고 "성공할 것"이라 가정해 UI를 즉시 바꾸고, 실패하면 되돌리는(rollback) 패턴입니다. 좋아요·즐겨찾기·토글처럼 가볍고 되돌리기 쉬운 동작에 체감 속도를 키워줍니다. 아래 ❤️는 누르면 즉시 켜지고, 1초 뒤 서버 응답을 흉내 냅니다 — 토글로 실패를 강제하면 자동 롤백 + 에러 토스트가 뜹니다.

왜 이렇게? 클릭 → 즉시 카운트 +1(낙관적) → 서버 실패 시 원래 값으로 복구 + 알림. 사용자는 평소엔 빠름을 체감하고, 드문 실패만 인지합니다.

⚠️ 결제엔 함부로 쓰지 말 것. "결제 성공"처럼 되돌릴 수 없거나 돈이 오가는 상태는 낙관적으로 미리 보여주면 안 됩니다. 사용자가 성공으로 오해할 수 있어서요. paysable에선 좋아요·필터·정렬·메모 같은 보조 동작에만 쓰고, 승인 결과는 반드시 서버 확정 후 표시합니다.
