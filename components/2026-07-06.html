셀렉트 & 콤보박스 ⌨️ 입력·폼

선택지가 7개 이하면 네이티브 <select>가 정답 — 모바일 휠 UI·접근성·무료 키보드 동작을 공짜로 얻는다. 선택지가 많고 검색이 필요할 때만 콤보박스(입력창+목록)를 직접 만든다. 은행·국가처럼 수십 개 항목이 그렇다.

화살표만 커스텀하고 열림·키보드·모바일 동작은 브라우저에 맡긴다. 가장 안전한 선택.
    ↓↑ 이동 · Enter 선택 · Esc 닫기. 입력한 글자는 강조된다.

    • 언제 무엇을: 옵션 소수 → select. 옵션 다수 + 검색 필요 → 콤보박스. "예뻐 보이려" 커스텀 드롭다운을 남발하면 접근성·모바일에서 손해다.
    • 콤보박스 = ARIA 계약: 입력창은 role="combobox", 목록은 role="listbox", 항목은 role="option". 활성 항목은 aria-activedescendant로 가리켜 스크린리더가 읽게 한다. (ARIA = 보조기술에 의미를 알려주는 HTML 속성 규약)
    • 결과 개수를 알린다: aria-live 영역에 "3개 결과"를 넣어 눈으로 못 보는 사용자도 좁혀지는 걸 안다.
    <!-- 콤보박스의 뼈대: 역할(role)이 핵심 -->
<input role="combobox" aria-expanded="false" aria-controls="list"
       aria-autocomplete="list" autocomplete="off">
<ul role="listbox" id="list">
  <li role="option" id="opt-1" aria-selected="true">국민은행</li>
  <li role="option" id="opt-2">기업은행</li>
</ul>

    스위치 & 복사 버튼 🔘 버튼·액션

    두 개의 작은 액션 컴포넌트. 스위치는 "즉시 적용되는 on/off"에만 쓰고(저장 버튼이 따로 필요하면 체크박스가 맞다), 복사 버튼은 계좌번호·쿠폰처럼 손으로 옮겨적기 싫은 값에 붙여 실수를 없앤다.

    토글 스위치 — 켜는 즉시 반영

    role="switch" + <input type=checkbox> = 스페이스바 토글·포커스 링을 공짜로 얻는다. 알림 ON · sandbox OFF

    복사 버튼 — 클릭하면 "복사됨"으로 잠깐 변신

    110-234-567890
    아이콘·글자가 ✓ 복사됨으로 1.5초 바뀐 뒤 원복. aria-live로 스크린리더도 "복사됨"을 듣는다.

    • 스위치 vs 체크박스: 스위치는 즉시 적용(설정이 바로 켜짐), 체크박스는 나중에 저장(폼 제출 시 반영). 저장 버튼이 있는 폼 안이라면 스위치는 오해를 부른다.
    • 낙관적 UI가 아니다: 결제 관련 스위치(sandbox 전환 등)는 서버 확정 전 켜진 척하지 말 것. 실패 시 원복 애니메이션이 사용자에게 불신을 준다.
    • 복사 성공은 두 채널로: 시각(아이콘 변화) + 청각(aria-live). 색만으로 성공을 표시하면 색각 이상 사용자가 놓친다.
    • reduced-motion 존중: prefers-reduced-motion(모션 최소화 설정)이면 썸(thumb) 슬라이드 전환을 끈다.
    // 복사 + 성공 피드백 (핵심만)
async function copy(btn){
  await navigator.clipboard.writeText(btn.dataset.copy);
  btn.classList.add('done');
  btn.querySelector('.lbl').textContent = '복사됨';
  status.textContent = '계좌번호가 클립보드에 복사됐어요';  // aria-live 낭독
  setTimeout(() => { btn.classList.remove('done');
    btn.querySelector('.lbl').textContent = '복사'; }, 1500);

    field-sizing: content — 입력창이 내용만큼 자라는 CSS 🌀 트렌드·개념

    field-sizing: content는 <textarea>·<input>이 내용 길이에 맞춰 스스로 커지고 줄어드는 CSS 한 줄이다. 예전엔 "scrollHeight 재서 height 다시 세팅"하는 JS를 붙였는데, 이제 CSS 한 줄로 끝난다. 2026년 들어 주요 브라우저에 두루 퍼졌다(Baseline).

    field-sizing:content + max-height로 상한을 두면 무한정 커지지 않고 그 뒤엔 스크롤로 전환된다.
    ₩
    숫자 폭만큼만 차지
    금액·태그·인라인 편집처럼 "값 길이에 딱 맞는 폭"이 필요한 곳에서 레이아웃이 깔끔해진다.
    • 왜 지금 뜨나: 오랫동안 auto-grow textarea는 "JS로 높이 계산" 단골 문제였다. field-sizing이 2026년 Baseline에 들며 JS·라이브러리 없이 해결 가능해졌다.
    • 상·하한을 꼭 건다: min-height/max-height(또는 input은 min-width/max-width)로 경계를 두지 않으면 레이아웃이 튄다.
    • 점진적 향상(progressive enhancement): 이 속성을 모르는 옛 브라우저는 그냥 무시하고 기본 입력창으로 보인다 — 깨지지 않으니 지금 바로 얹어도 안전하다.
    • 추정: 세부 지원 버전은 브라우저마다 다를 수 있으니 실제 배포 전 caniuse.com에서 대상 브라우저를 확인하는 걸 권장.
    /* 이것이 전부다 — JS 0줄 */
textarea{
  field-sizing: content;   /* 내용에 맞춰 높이 자동 */
  min-height: 2.6em;
  max-height: 9em;         /* 여기부턴 스크롤 */
}
input.amount{
  field-sizing: content;   /* 값 길이에 맞춰 폭 자동 */
  min-width: 2ch; max-width: 12ch;
}
