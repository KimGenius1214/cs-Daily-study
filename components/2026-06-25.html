🧭 내비게이션·탭 탭 & 세그먼트 컨트롤 — 슬라이딩 인디케이터와 키보드 동작

요즘 탭은 선택된 항목 아래로 표시자(인디케이터)가 미끄러지듯 이동한다. 핵심은 보기보다 접근성 구조다: 탭 묶음은 role="tablist", 각 탭은 role="tab", 화살표 키로 이동하는 로빙 탭인덱스(roving tabindex: 묶음 안에서 Tab 키로 들어오는 진입점은 하나, 안에서는 ←→로 이동)를 쓴다.

변형 A — 알약형(pill) 탭. 결제 대시보드의 기간 전환에 잘 맞음

오늘 승인 결제 1,284건 · 거래액 ₩38.2M
이번 주 승인 결제 8,901건 · 거래액 ₩264M
이번 달 승인 결제 37,420건 · 거래액 ₩1.12B

클릭해 보고, 탭을 한 번 누른 뒤 키보드 ← → 로도 이동해 보세요. 인디케이터가 따라 움직입니다.

변형 B — 밑줄형(underline) 탭. 화면 폭이 좁거나 콘텐츠 영역 상단에 적합

결제 수단·승인율·평균 거래액을 한눈에.
정산 주기와 다음 입금 예정일.
최근 환불 요청과 처리 상태.

밑줄 인디케이터도 같은 로직(선택 탭의 위치·너비를 측정해 이동)으로 동작합니다.

핵심 코드 보기 (키보드 + 인디케이터) 
// roving tabindex + 화살표 이동 + 인디케이터 측정
function setupTabs(list){
  const tabs=[...list.querySelectorAll('[role=tab]')];
  const ind=list.querySelector('.tab-indicator');
  const move=el=>{ ind.style.left=el.offsetLeft+'px'; ind.style.width=el.offsetWidth+'px'; };
  function select(el){
    tabs.forEach(t=>{
      const on=t===el;
      t.setAttribute('aria-selected',on);
      t.tabIndex=on?0:-1;                       // 선택된 것만 Tab 진입점
      document.getElementById(t.getAttribute('aria-controls')).hidden=!on;
    });
    move(el);
  }
  list.addEventListener('click',e=>{ const t=e.target.closest('[role=tab]'); if(t) select(t); });
  list.addEventListener('keydown',e=>{
    const i=tabs.indexOf(document.activeElement); if(i<0) return;
    let n; if(e.key==='ArrowRight') n=(i+1)%tabs.length;
    else if(e.key==='ArrowLeft') n=(i-1+tabs.length)%tabs.length;
    else if(e.key==='Home') n=0; else if(e.key==='End') n=tabs.length-1; else return;
    e.preventDefault(); tabs[n].focus(); select(tabs[n]);   // 포커스 이동과 동시에 선택
  });
  move(tabs.find(t=>t.getAttribute('aria-selected')==='true'));
}

🌀 트렌드·개념 JS 없는 메뉴 — Popover API + CSS Anchor Positioning

Popover API(popover 속성): 버튼에 popovertarget만 달면 열고/닫고, ESC·바깥 클릭 닫힘, 최상위 레이어 쌓임을 브라우저가 처리한다(2024년 Baseline 진입). CSS Anchor Positioning: 팝오버를 특정 요소에 붙여서 위치시키는 CSS 기능 — 자리 계산용 JS 라이브러리가 필요 없다(2026년 Baseline 진입, 일부 플립 세부는 최신 버전 필요, 추정).

버튼을 누르면 메뉴가 버튼에 붙어 열립니다. ESC 또는 바깥 클릭으로 닫혀요 — 위치/닫힘 모두 JS 없이 동작.

⚠️ 이 브라우저는 CSS Anchor Positioning 미지원으로 보입니다. 팝오버는 열리지만 위치가 기본값일 수 있어요. (실서비스에선 @supports로 폴백 위치를 줍니다.)

핵심 코드 보기 (HTML + CSS만) 
<!-- HTML: JS 0줄 -->
<button popovertarget="acct-menu" style="anchor-name:--acct-btn">계정 ▾</button>
<div id="acct-menu" popover role="menu">…</div>

/* CSS: 버튼(--acct-btn)에 메뉴를 붙임 */
#acct-menu{
  position:absolute;
  position-anchor:--acct-btn;          /* 이 앵커를 기준으로 */
  top:anchor(bottom); left:anchor(left); /* 버튼 아래-왼쪽 정렬 */
  margin-top:8px;
  position-try-fallbacks:flip-block;   /* 아래 공간 없으면 위로 뒤집기 */
}

/* 미지원 브라우저 폴백 */
@supports not (anchor-name:--x){
  #acct-menu{ position:absolute; top:48px; left:0; }
}

📈 왜 지금 뜨나: Floating UI 같은 위치 계산 라이브러리(수십 KB)를 걷어내고, 드롭다운·툴팁·메뉴를 플랫폼 기본 기능으로 만드는 흐름. 접근성(포커스 트랩·ESC·라이트 디스미스)을 브라우저가 보장해 손으로 짜던 버그가 줄어든다.
