🔘 분할 버튼 · 메뉴 버튼 (Split / Menu Button) 버튼·액션

가장 흔한 기본 동작 하나를 바로 실행하게 두고, 덜 쓰는 변형들은 드롭다운 메뉴 뒤에 숨기는 패턴. 버튼 하나에 선택지를 다 늘어놓지 않아 화면이 깔끔해진다. 메뉴는 키보드로 완전히 조작된다(↑↓·Home·End·Esc).

← 왼쪽은 즉시 실행, 오른쪽 ▾ 는 메뉴

주 동작(저장)은 한 번의 클릭. 캐럿(▾)을 누르거나 포커스 후 ↓ 를 누르면 메뉴가 열린다.

행 액션:

아이콘만 있는 메뉴 버튼(⋯). 카드·테이블 행처럼 공간이 좁을 때. 메뉴는 오른쪽 정렬로 화면 밖으로 안 새게.

아직 실행된 동작 없음

  • aria-haspopup/aria-expanded로 "여는 버튼"임을 스크린리더에 알린다. 메뉴는 role="menu", 항목은 role="menuitem".
  • 키보드: 포커스 상태에서 ↓ 로 열고 첫 항목에 포커스, ↑↓ 이동, Home/End 처음·끝, Esc 닫고 버튼으로 복귀.
  • 바깥 클릭이나 포커스 이탈 시 닫힌다. 메뉴가 화면 밖으로 나가지 않도록 정렬 방향을 상황에 맞게(여기선 CSS로, 실무에선 앵커/Floating UI로).

⌨️ 콤보박스 · 자동완성 (Combobox) 입력·폼

선택지가 수십~수백 개일 때 <select> 대신 쓰는 패턴. 타이핑하면 목록이 좁혀지고, 입력값과 겹치는 글자는 하이라이트된다. 결제에서 통화·은행·국가 선택에 딱 맞는다.

    선택된 통화: 없음

    • 콤보박스 = 텍스트 입력 + 펼침 목록이 합쳐진 컨트롤. role="combobox" 입력과 role="listbox" 목록을 aria-controls로 연결한다.
    • aria-activedescendant: 포커스는 입력창에 그대로 두고, "지금 가리키는 항목"의 id만 이 속성으로 알린다. 그래서 타이핑하며 ↑↓ 로 목록을 훑을 수 있다.
    • 키보드: ↓ 열기·다음, ↑ 이전, Enter 선택, Esc 닫기. 입력 즉시 필터링(대소문자·초성 아닌 부분일치).
    • 결과 0건일 때 "검색 결과 없음"을 명확히 보여준다(빈 상태 설계).

    🌀 CSS 앵커 포지셔닝 — JS 없이 붙는 팝오버 트렌드·개념

    앵커 포지셔닝: 어떤 요소를 다른 요소(앵커)에 CSS만으로 "붙여" 배치하는 새 표준. 지금까지 툴팁·드롭다운·팝오버 위치는 JS 라이브러리(Floating UI 등)로 매 프레임 계산했는데, 그걸 CSS가 대신한다. Baseline 2026으로 올라왔다(Chrome 125+, Safari 26/18.2+, Firefox 최신). — 지원 사실 출처: MDN·web.dev.

    출금 가능 잔액은 미확정 결제 3건을 제외한 금액입니다. 위치는 CSS anchor() 로만 계산됨.
    확인 중… 이 브라우저의 앵커 포지셔닝 지원 여부를 @supports 로 감지합니다.

    버튼을 누르면 팝오버가 버튼 아래·가로 중앙에 붙는다. 좌표를 JS로 넣지 않았고, 자리 없으면 flip-block 으로 위로 뒤집힌다.

    • anchor-name:--apBtn 로 기준 요소에 이름을 붙이고, 팝오버에서 position-anchor:--apBtn + top:anchor(bottom) 로 "그 요소의 아래"에 놓는다.
    • position-try-fallbacks(예: flip-block): 아래 공간이 부족하면 자동으로 위로 뒤집는 대체 위치. 스크롤·경계 처리를 브라우저가 맡는다.
    • 미지원 브라우저 대비: @supports (anchor-name:--x) 로 감싸고, 아니면 기존 방식(JS 라이브러리/일반 absolute)으로 폴백. 이 데모도 그렇게 감쌌다.
