⌨️ 텍스트 필드 — 플로팅 라벨 · 인라인 검증 · 비밀번호 토글 입력·폼

2026년 폼 입력의 기본기. 라벨을 placeholder처럼 겹쳐 두다가 입력하면 위로 떠오르는 플로팅 라벨, 사용자가 건드린 뒤에만 빨개지는 인라인 검증, 그리고 비밀번호 가림/보임 토글을 한 번에.

예: name@ableunion.com

플로팅 라벨 — 빈 칸일 땐 라벨이 placeholder 자리에 있다가, 포커스하거나 값이 차면 위로 올라갑니다. 라벨이 항상 살아있어 접근성·공간 둘 다 잡습니다.

숫자 4자리를 입력하세요

:user-invalid — 사용자가 입력을 건드리고 떠난 뒤에만 빨개집니다. 페이지 로드 직후부터 빨갛게 다그치지 않아 덜 공격적입니다. Baseline 2023.10~

8자 이상

비밀번호 토글 — 오타로 인한 로그인 실패를 줄입니다. 버튼은 aria-pressed로 상태를 읽어주고, 포커스는 입력칸에서 빼앗지 않습니다.

입력하면 천 단위로 자동 구분됩니다

입력 중 포맷팅 — 타이핑하면 1,250,000처럼 콤마가 붙습니다. 커서를 끝에 두고 숫자만 다시 깔아 자리수를 헷갈리지 않게 합니다.

용어 — :focus-visible: 키보드로 이동했을 때만 포커스 링을 보여주는 셀렉터(마우스 클릭엔 안 뜸). :user-invalid: 사용자가 한 번 상호작용한 뒤 값이 규칙에 안 맞을 때만 켜지는 셀렉터.

🔘 위험 액션 확인 — 더블 컨펌 · 길게 눌러 실행 · 실행취소 버튼·액션

환불·삭제·해지처럼 되돌리기 힘든 액션을 안전하게 만드는 세 가지 패턴. "정말 하시겠어요?" 모달 남발 대신, 마찰을 적절히 주거나 되돌릴 길을 줍니다.

더블 컨펌(인라인) — 한 번 누르면 "한 번 더 누르면 취소돼요"로 바뀌고, 3초 안에 안 누르면 원래대로. 모달을 띄우지 않고도 실수 클릭을 막습니다.

길게 눌러 실행(hold-to-confirm) — 0.9초간 누르고 있어야 발동. 누르는 동안 버튼이 차오릅니다. 모바일·터치에서 "확인" 모달보다 빠르고 직관적입니다.

삭제했습니다.

실행취소(undo) — 막지 말고 즉시 실행하되 5초간 되돌릴 창을 줍니다. 가장 마찰이 적은 방식 — 대부분의 사용자는 한 번에 끝내고, 실수한 사람만 되돌립니다.

어떤 걸 언제?

· 실행취소: 되돌릴 수 있는 일(목록에서 숨기기, 보관) — 1순위.
· 더블 컨펌: 화면을 안 떠나고 빠르게 막고 싶을 때.
· 꾹 눌러: 터치 환경 + 진짜 위험(환불 확정).
· 모달: 추가 입력·약관 동의가 필요할 때만(과용 금지).

용어 — hold-to-confirm: 버튼을 일정 시간 누르고 있어야 실행되는 패턴. optimistic(낙관적) undo: 서버 응답을 기다리지 않고 화면을 먼저 바꾸되, 되돌릴 버튼을 함께 띄우는 방식. 단, 결제 확정처럼 실패 시 위험한 작업엔 서버 확인 후에 처리합니다.

🌀 컨테이너 쿼리 — 화면이 아니라 "부모 폭"에 반응하는 컴포넌트 트렌드·개념

미디어 쿼리는 뷰포트(브라우저 창) 폭만 봅니다. 컨테이너 쿼리는 컴포넌트가 놓인 부모 요소의 폭을 봅니다. 같은 카드를 사이드바에 넣든 본문에 넣든, 제자리 폭에 맞춰 알아서 바뀝니다. 디자인 시스템·컴포넌트 단위 개발과 궁합이 좋아 2026년 표준 도구가 됐습니다.
