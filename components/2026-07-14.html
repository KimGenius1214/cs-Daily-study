🪟 바텀시트 · 드로어 (Bottom Sheet) 오버레이·모달·토스트

모바일 결제·확인 흐름에서 화면 아래에서 슬라이드로 올라오는 시트. 모달 다이얼로그의 모바일 변형으로, 드래그 핸들·스크림(어두운 배경)·아래로 밀어 닫기를 갖춘다.

① 결제 확인 시트

버튼을 누르면 아래에서 올라옵니다. 스크림 클릭 · Esc · 아래로 드래그로 닫힘.

② 왜 모달 대신 시트인가

모바일에서 화면 중앙 모달은 엄지 도달이 어렵다. 하단 시트는 한 손 엄지 영역에 액션 버튼을 두고, 물리적 제스처(아래로 밀기)로 닫기를 자연스럽게 만든다. 결제·필터·공유 시트에 표준.

💡 드래그 핸들(grip): 상단 짧은 회색 막대. "이건 잡아서 움직일 수 있다"는 어포던스(사용자가 조작 방법을 직관으로 알게 하는 시각 힌트) 신호다.

🔔 진행률 표시 · 스텝퍼 (Progress & Stepper) 피드백·상태(로딩·빈상태·에러)

"얼마나 남았나"를 알려주는 두 방식. 남은 양을 아는 작업엔 결정형(determinate) 막대, 모르는 대기엔 비결정형(indeterminate) 막대. 여러 단계 흐름엔 스텝퍼.

① 결정형 진행률

업로드·다운로드처럼 %를 아는 작업.

0%

② 비결정형 진행률

서버 응답 대기처럼 끝을 모를 때. 값 대신 "진행 중"만 전달.

계속 흐르는 애니메이션 = "멈춘 게 아니라 처리 중"

③ 다단계 스텝퍼 — 체크아웃 진행

지금 어디고, 몇 단계 남았는지 한눈에. 결제 3~4단계 흐름의 표준.

💡 왜 role="progressbar"? 스크린리더에 "진행 막대, 현재 40%"로 읽힌다. 결정형은 aria-valuenow를 갱신, 비결정형은 값을 빼고 aria-label만 둔다.

🌀 View Transitions API — 상태가 바뀔 때 부드럽게 트렌드·개념

View Transitions API: DOM이 바뀌는 순간을 브라우저가 스냅샷으로 찍어 자동으로 크로스페이드·이동 애니메이션을 넣어주는 표준 API. 리스트 → 상세 전환 같은 걸 라이브러리 없이 네이티브 앱처럼 매끄럽게.

확인 중… document.startViewTransition 지원 여부를 감지합니다.
