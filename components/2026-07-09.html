바텀 내비게이션 & 탭바 🧭 내비게이션·탭

모바일 앱의 하단 고정 내비는 엄지가 닿는 자리에 핵심 5개 이하를 놓는 패턴이다. 핀테크 앱(홈·거래·송금·카드·더보기)의 사실상 표준. 핵심은 현재 위치 표시(aria-current), 배지 알림, 그리고 노치·홈바를 피하는 safe-area 여백이다.

홈

이번 달 결제 3건 · 예정 출금 1건

홈 거래 송금 카드 3 더보기

아이콘을 눌러 보세요 — 화면 제목과 현재 위치(파란색)가 바뀝니다.

  • 현재 위치는 aria-current="page": 색·굵기만으로 표시하면 스크린리더는 "지금 어디"인지 모른다. 이 속성이 "현재 페이지"라고 낭독하게 한다.
  • 탭 개수는 5개 이하: 그 이상은 마지막을 "더보기"로 묶는다. 손가락 터치 타깃은 최소 44×44px 권장.
  • safe-area: env(safe-area-inset-bottom)(노치·홈 인디케이터 영역 크기를 알려주는 CSS 값)만큼 아래 패딩을 줘 홈바에 안 가리게 한다.
  • 배지는 숫자 + 색 + 위치로 알린다. 색만으로는 색각 이상 사용자가 놓친다.
<nav aria-label="주요 메뉴">
  <a href="/home" aria-current="page">홈</a>   <!-- 현재 위치 -->
  <a href="/tx">거래</a>
</nav>

.bnav{ padding-bottom: calc(6px + env(safe-area-inset-bottom)); } /* 홈바 회피 */
.bnav a[aria-current="page"]{ color: var(--brand); }             /* 활성 표시 */

아코디언 리스트 — 접었다 펴는 disclosure 🃏 카드·리스트

긴 정보를 제목만 보여주고, 누르면 상세가 펼쳐지게 접는 패턴(disclosure). FAQ·약관·주문 상세에 흔하다. 요즘은 JS 없이 <details>/<summary>로 만들고, "한 번에 하나만 열기"만 살짝 스크립트로 제어한다.

결제가 실패했어요. 왜인가요?
한도 초과, 카드 정지, 잔액 부족이 대부분입니다. 결제 내역에서 실패 사유 코드를 확인하고, 카드사에 문의하거나 다른 수단으로 다시 시도하세요.
환불은 얼마나 걸리나요?
신용카드 기준 영업일 3~5일이 일반적입니다. 카드사 매입 취소 시점에 따라 명세서 반영이 다음 달로 넘어갈 수 있습니다.
정기결제를 해지하려면?
설정 > 구독 관리에서 해지할 수 있습니다. 다음 결제일 전까지 해지하면 추가 청구가 없습니다.

제목을 누르면 하나만 열립니다(다른 건 자동으로 닫힘). 지원 브라우저에선 높이가 부드럽게 전환됩니다.

  • 기본은 네이티브: <details>는 키보드 포커스·엔터 토글·열림 상태를 공짜로 준다. "한 번에 하나만" 규칙만 JS로 얹는다.
  • 부드러운 높이: 예전엔 height:auto로는 트랜지션이 안 걸렸다. 이제 ::details-content(펼쳐지는 내용 부분을 가리키는 가상 요소)에 interpolate-size로 애니메이션한다 — 아래 세 번째 항목에서 자세히.
  • 마커 커스텀: 기본 삼각형은 summary::-webkit-details-marker{display:none}로 숨기고 직접 셰브런(∨)을 넣어 회전시킨다.
// "한 번에 하나만 열기" (핵심만)
const items = group.querySelectorAll('details');
items.forEach(d => d.addEventListener('toggle', () => {
  if(d.open) items.forEach(o => { if(o!==d) o.open = false; });
}));

interpolate-size — height: auto로 부드럽게 여닫기 🌀 트렌드·개념

오랜 숙제였던 "height: 0 → height: auto 애니메이션"을 드디어 CSS만으로 푼다. 루트에 interpolate-size: allow-keywords 한 줄이면, auto·min-content 같은 키워드 크기로도 트랜지션이 걸린다.

버튼을 눌러 접었다 펴 보세요 — 내용 높이를 몰라도 height:auto로 부드럽게 전환됩니다(지원 브라우저).

⚠️ 지원 현황(2026-07 기준): Chromium 계열(Chrome/Edge 129+)만 지원, Firefox·Safari 미지원. 점진적 향상으로 쓰면 안전 — 미지원 브라우저는 애니메이션 없이 즉시 열림/닫힘으로 동작(기능은 그대로).
  • 무엇을 푸나: 지금까진 내용 높이를 JS로 재거나(scrollHeight), grid-template-rows: 0fr→1fr 트릭을 썼다. interpolate-size는 그걸 대체한다.
  • 쓰는 법 두 가지: ① 루트에 interpolate-size: allow-keywords 선언(상속됨). ② 또는 calc-size(auto)를 값에 쓰면 그 자리에서 자동 활성화.
  • 점진적 향상(progressive enhancement): "되는 브라우저에서만 애니메이션, 안 되는 곳도 멀쩡하게" 두는 전략. 지금 얹어도 아무것도 깨지지 않는다.
  • 추정: 지원 버전은 계속 바뀌니 배포 전 caniuse.com에서 대상 브라우저 확인 권장.
/* 방법 1 — 문서 전체에 허용 (상속) */
:root{ interpolate-size: allow-keywords; }
.panel{ height: 0; overflow: hidden; transition: height .28s ease; }
.panel[data-open]{ height: auto; }   /* 이제 auto로도 트랜지션됨 */

/* 방법 2 — 필요한 곳에서만 calc-size로 자동 활성화 */
.panel{ transition: height .28s; }
.panel.open{ height: calc-size(auto, size); }
