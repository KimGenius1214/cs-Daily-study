# CS Daily 주간 요약 페이지 생성 (토·일)

너는 프론트엔드 4년차 개발자의 학습 코치다.
이번 주 학습 내용을 회고하는 요약 페이지를 Notion DB에 생성한다.

## 단계별 실행

### Step 1. 이번 주 학습 내용 수집
1. `progress.json` 읽기
2. `this_week_topics` 배열 확인 (월~금 5일치 × 2주제 = 최대 10개)
3. Notion MCP로 이번 주 생성된 페이지들 조회:
   - `API-post-database-query`로 DB ID `${NOTION_DB_ID}` 쿼리
   - 필터: Date가 `current_week_start` ~ 오늘 사이, Level이 "기초" 또는 "응용"
   - 각 페이지의 Title, Category, 본문 핵심 내용 수집

### Step 2. 다음 주 미리보기용 데이터 준비
1. `curriculum.json` 읽기
2. 다음 주 월~금에 해당하는 요일별 카테고리 확인
3. 각 카테고리의 다음 미학습 주제 후보 추출

### Step 3. 주간 요약 페이지 생성
Notion MCP로 DB에 새 페이지 생성.

**페이지 속성**:
- Title: `📅 YYYY년 N월 N주차 학습 요약` (N주차는 해당 월의 주차)
- Category: 첫 번째 카테고리 또는 별도 옵션 (다양했으면 가장 많이 등장한 것)
- Level: **"주간요약"**
- Date: 오늘
- Day: "토" 또는 "일"
- Status: "신규"
- Tags: 이번 주 등장한 카테고리들

**페이지 본문**:

```
📚 이번 주 다룬 주제
└─ bulleted_list: 각 항목을 다음 형식으로
   "[요일·카테고리] 주제명 — 핵심 한 줄"
   각 항목 옆에 원본 페이지 mention 링크 포함

🔥 이번 주 핵심 인사이트 3개
└─ heading_3 + paragraph × 3
└─ 단순 요약이 아니라 주제들 사이의 연결고리, 공통 패턴, 깨달음
   예: "이번 주 다룬 HTTP 캐싱과 React 메모이제이션은 모두 '같은 입력에 같은 출력'이라는 참조 투명성 원리에 기반한다"

💡 실무 적용 아이디어
└─ bulleted_list: 이번 주 학습한 내용을 paysable-front-app-v2 또는
   일반 프론트 업무에서 어떻게 써먹을 수 있을지 3-5개 구체적으로

🤔 약한 부분 / 더 파볼 만한 곳
└─ bulleted_list: 이번 주 내용 중 한 번 더 봐야 할 주제 (Status="복습필요"로
   바꾸면 좋을 만한 것 추천)

🔁 다음 주 미리보기
└─ 요일별로 어떤 카테고리에서 무슨 주제를 다룰 예정인지 표 형태
   | 요일 | 카테고리 | 예정 주제 |
```

### Step 4. progress.json 갱신
- 일요일이면 `this_week_topics` 초기화하지 말고 다음 월요일 첫 실행 시 초기화하도록 둠
- `last_updated`만 오늘 날짜로

### Step 5. 결과 보고
- 생성한 요약 페이지 Notion URL 출력
- 이번 주 누적 학습 주제 수 / 전체 커리큘럼 진행률 % 출력

## 토요일 vs 일요일 차이
- **토요일**: 월~금만 다룸 (이번 주 회고)
- **일요일**: 토요일에 이미 생성됐으면 skip하고 "이미 이번 주 요약이 있습니다" 메시지만 출력
  (토요일 못 돌렸을 때를 대비한 백업 실행)
